Swing-by service

Intended to help in using Firefox's insanely great Tab Container feature.

If you point to this URL: ...

  • this.url/some-word/twitter.com/flypenguin_de

... it will redirect you to ...

  • twitter.com/flypenguin_de

... ignoring some-word, and keeping everything else (including query strings).

Why is this useful?

The usefulness lies in the some-word part of the original url, and the Firefox addon Containerise. Generally you can now configure any this.url/some-word/... prefix to be opened in a specific tab container, so you can open the same web service (e.g. Twitter, AWS, ...) with different accounts in parallel, without any hassle.

What? How?

Let's get concrete: Let's say, you want to separate some work-related stuff from your private browsing, without opening two browsers. You simply need to save the following bookmarks:

  • this.url/business/gmail.com
  • this.url/business/12345678.signin.aws.amazon.com/console/
  • this.url/business/teams.microsoft.com (oh that one is so useful!)

Now, you configure Containerise to open this.url/business/* in the tab container "business". In that container, you can log in to all those services â€“ and those logins will not spill over into your "normal" browser tabs.

Or, if you're a freelancer, you could do this:

  • this.url/client1/teams.microsoft.com
  • this.url/client2/teams.microsoft.com
  • this.url/client3/teams.microsoft.com

Then simply save them as bookmarks, log in with the respective accounts in the tab containers, and you have fully isolated logins available at your fingertips.

If you now assign shortcuts to those bookmarks (e.g. cl1-teams, ...) then you can simply enter "cl1-teams" into Firefox's address bar and open Microsoft Teams for "client1", logged in and ready, "just like that". It does not get more convenient than this.
